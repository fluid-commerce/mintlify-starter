Fluid exposes data to your Liquid templates through a set of variables that are automatically populated at render time. Some variables are available on every page regardless of type; others are scoped to specific page contexts and only exist when the corresponding content is being rendered. Understanding which variables are available where will help you write templates that are both powerful and reliable.

## Global Variables

These variables are available in every template, every layout, and every section â€” no matter which page type is being rendered.

### `company.*`

Information about the current storefront company.

| Variable | Description |
|---|---|
| `company.name` | The company's display name |
| `company.logo_url` | URL of the company logo image |
| `company.shop_page_url` | URL to the main shop page |
| `company.checkout_url` | URL to the checkout flow |
| `company.points_label.singular` | The singular label for the loyalty points currency (e.g. "Point") |
| `company.points_label.plural` | The plural label for the loyalty points currency (e.g. "Points") |

### `request.*`

Information about the current HTTP request.

| Variable | Description |
|---|---|
| `request.path` | The URL path of the current page (e.g. `/products/my-product`) |
| `request.host` | The hostname of the request (e.g. `mystore.fluid.app`) |
| `request.page_type` | The template type being rendered (e.g. `product`, `shop_page`) |
| `request.query_parameters` | A map of query string parameters |

### `affiliate.*`

Information about the affiliate context, when the page is viewed through an affiliate link.

| Variable | Description |
|---|---|
| `affiliate.name` | The affiliate's display name |
| `affiliate.my_site_url` | The affiliate's personal storefront URL |
| `affiliate.sign_in_url` | URL to the affiliate sign-in page |
| `affiliate.sign_out_url` | URL to the affiliate sign-out page |

### `localization.*`

The visitor's active locale settings.

| Variable | Description |
|---|---|
| `localization.country` | The visitor's detected or selected country |
| `localization.language` | The active language object (includes `.iso_code`) |

### Global Arrays

| Variable | Description |
|---|---|
| `products[]` | Array of featured or recently viewed products, each with `title`, `url`, `price`, and `images[]` |
| `collections[]` | Array of all available collections |
| `enrollment_packs[]` | Array of all active enrollment packs |

## Page-Specific Scopes

The following variables are only populated when the corresponding page type is being rendered. Accessing them outside their scope returns an empty value.

| Page Type | Key Variables |
|---|---|
| **Product** | `product.name`, `product.price`, `product.images`, `product.selected_or_first_available_variant` |
| **Shop / Collection** | `collection.filters`, `collection.products`, `pagination` |
| **Home** | `products[]`, `enrollment_packs[]` |
| **Media** | `media.title`, `media.embed_url`, `media.poster` |
| **Enrollment Pack** | `enrollment_pack.title`, `enrollment_pack.membership_products` |
| **Cart** | `cart.items`, `cart.total` |
| **Join** | `collection.enrollment_packs` |

## Using Points Labels

Fluid allows each company to rename their loyalty currency to something that fits their brand â€” for example, "Make Rewards" instead of "Points." The `company.points_label` object always reflects whatever label the company has configured, so your templates stay accurate automatically.

Use these variables anywhere you display a loyalty points balance or description:

```liquid
<p>
  You have {{ customer.points_balance }}
  {{ customer.points_balance == 1
    ? company.points_label.singular
    : company.points_label.plural }}.
</p>
```

For simpler inline use:

```liquid
{{ product.points }} {{ company.points_label.plural }}
```

This way, if a company renames their currency from "Points" to "Coins" or "Stars", every template that uses `company.points_label` updates automatically without any code changes.

## Defensive Coding

Because page-specific variables are only populated in their respective scopes, it is good practice to guard against empty values â€” especially in shared components and sections that may be included across multiple page types.

Use the `| default` filter to provide a fallback for any variable that might be absent:

```liquid
<title>{{ page_title | default: company.name }}</title>
<img src="{{ product.images[0] | img_url: '600x400' | default: 'placeholder.png' | asset_url }}" />
```

Wrap optional objects in an `{% if %}` guard before accessing their properties:

```liquid
{% if affiliate %}
  <p>Shopping with {{ affiliate.name }}</p>
{% endif %}

{% if product %}
  <span>{{ product.price | money }}</span>
{% endif %}
```

<Warning>
  Referencing a page-scoped variable outside its intended page type â€” for example, using `product.price` in a footer section â€” will render as empty without raising an error. If you notice missing content, check that the variable you are accessing is in scope for the current page type.
</Warning>
