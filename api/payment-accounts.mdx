Payment accounts are Fluid's representation of your connections to external payment processors and PSPs (Payment Service Providers). Each payment account holds the credentials and configuration needed for Fluid to route charges through that processor. You can connect multiple accounts — such as Stripe, Braintree, or PayPal — and then use the Payment Routing API to distribute traffic across them.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/payment_accounts` | List payment accounts |
| `POST` | `/api/payment_accounts` | Create a payment account |
| `GET` | `/api/payment_accounts/{id}` | Get a payment account |
| `PUT` | `/api/payment_accounts/{id}` | Update a payment account |
| `DELETE` | `/api/payment_accounts/{id}` | Delete a payment account |
| `GET` | `/api/payment_accounts/credentials_schema` | Get credentials schema |
| `GET` | `/api/payment_accounts/available_countries` | Countries by processor |
| `GET` | `/api/payment_integrations` | List integrations |
| `GET` | `/api/payment_integrations/{integration_class}` | Get integration details |

## List Payment Accounts

Retrieve all payment accounts connected to your company. The response includes each account's processor type, display name, active status, and supported currencies.

```bash
curl https://api.fluid.app/api/payment_accounts \
  -H "Authorization: Bearer <token>"
```

**Response**

```json
{
  "payment_accounts": [
    {
      "id": "acc_01HXYZ",
      "name": "Stripe Production",
      "integration_class": "stripe",
      "active": true,
      "currencies": ["USD", "EUR", "GBP"],
      "created_at": "2025-01-15T10:00:00Z"
    },
    {
      "id": "acc_01HABC",
      "name": "Braintree US",
      "integration_class": "braintree",
      "active": true,
      "currencies": ["USD"],

    }
  ],
  "meta": {
    "request_id": "req_01HDEF",
    "timestamp": "2025-06-01T12:00:00Z"
  }
}
```

## Create a Payment Account

Connect a new processor to your Fluid account. Before creating an account, call the [credentials schema endpoint](#credentials-schema) to determine which fields are required for the target integration. Different processors require different credential fields (API keys, merchant IDs, secrets, etc.).

```bash
curl -X POST https://api.fluid.app/api/payment_accounts \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_account": {
      "name": "Stripe Production",
      "integration_class": "stripe",
      "active": true,
      "credentials": {
        "secret_key": "sk_live_xxxxxxxxxxxx",
        "publishable_key": "pk_live_xxxxxxxxxxxx"
      }
    }
  }'
```

<ParamField body="name" type="string" required>
  A display name for this payment account. Choose something descriptive, like `"Stripe EU"` or `"Braintree US"`.
</ParamField>

<ParamField body="integration_class" type="string" required>
  The integration identifier for the processor. Retrieve valid values from `GET /api/payment_integrations`.
</ParamField>

<ParamField body="active" type="boolean">
  Whether this account should be available for routing immediately. Defaults to `false`.
</ParamField>

<ParamField body="credentials" type="object" required>
  Processor-specific credential fields. The required keys vary by integration — call `GET /api/payment_accounts/credentials_schema?integration_class={class}` to get the exact schema before submitting.
</ParamField>

## Get a Payment Account

Retrieve details for a single payment account by its ID.

```bash
curl https://api.fluid.app/api/payment_accounts/acc_01HXYZ \
  -H "Authorization: Bearer <token>"
```

## Update a Payment Account

Update the name, credentials, or active status of an existing payment account. Use `PUT` to replace the full resource.

```bash
curl -X PUT https://api.fluid.app/api/payment_accounts/acc_01HXYZ \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_account": {
      "name": "Stripe Production (Updated)",
      "active": true,
      "credentials": {
        "secret_key": "sk_live_new_key_xxxxxxxxxxxx",
        "publishable_key": "pk_live_new_key_xxxxxxxxxxxx"
      }
    }
  }'
```

---

## Delete a Payment Account

Remove a payment account. This will fail if the account is referenced in an active routing configuration — deactivate or remove it from routing first.

```bash
curl -X DELETE https://api.fluid.app/api/payment_accounts/acc_01HXYZ \
  -H "Authorization: Bearer <token>"
```

## Credentials Schema

Before creating a payment account, retrieve the credentials schema for the target integration. This endpoint returns the exact field names, types, and validation rules required for that processor's `credentials` object — so you know exactly what to submit.

```bash
curl "https://api.fluid.app/api/payment_accounts/credentials_schema?integration_class=stripe" \
  -H "Authorization: Bearer <token>"
```

**Response**

```json
{
  "credentials_schema": {
    "integration_class": "stripe",
    "fields": [
      {
        "key": "secret_key",
        "label": "Secret Key",
        "type": "string",
        "required": true,
        "sensitive": true,
        "hint": "Starts with sk_live_ or sk_test_"
      },
      {
        "key": "publishable_key",
        "label": "Publishable Key",
        "type": "string",
        "required": true,
        "sensitive": false,
        "hint": "Starts with pk_live_ or pk_test_"
      }
    ]
  },
  "meta": {
    "request_id": "req_01HGHI",
    "timestamp": "2025-06-01T12:00:00Z"
  }
}
```

<Note>
  Fields with `sensitive: true` are write-only. Fluid will never return credential values in GET responses — only masked versions are shown.
</Note>

## Available Countries

Retrieve the list of countries supported by each processor. Use this to display region-appropriate processor options in your onboarding UI.

```bash
curl https://api.fluid.app/api/payment_accounts/available_countries \
  -H "Authorization: Bearer <token>"
```

## Payment Integrations

List all payment integrations available on the Fluid platform, or retrieve details on a specific one.

```bash
# List all integrations
curl https://api.fluid.app/api/payment_integrations \
  -H "Authorization: Bearer <token>"

# Get a specific integration
curl https://api.fluid.app/api/payment_integrations/stripe \
  -H "Authorization: Bearer <token>"
```

The integration detail response includes the processor's display name, logo URL, supported payment methods, supported currencies, and supported countries.

---

<ResponseField name="id" type="string">
  Unique identifier for the payment account.
</ResponseField>

<ResponseField name="name" type="string">
  Display name for the account as configured by you.
</ResponseField>

<ResponseField name="integration_class" type="string">
  The processor identifier (e.g. `stripe`, `braintree`, `paypal`).
</ResponseField>

<ResponseField name="active" type="boolean">
  Whether this account is available for payment routing.
</ResponseField>

<ResponseField name="currencies" type="array">
  List of currency codes this account is configured to accept.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the account was created.
</ResponseField>
