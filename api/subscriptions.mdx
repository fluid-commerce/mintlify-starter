The Subscriptions API lets you create and manage recurring order schedules for your customers. Use it to set up subscription plans, enroll customers, and control the subscription lifecycle — including pausing, resuming, cancelling, and reactivating subscriptions. Recurring orders are processed automatically according to the subscription plan's billing interval.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/subscriptions` | List subscriptions |
| `POST` | `/api/subscriptions` | Create a subscription |
| `GET` | `/api/subscriptions/{token}` | Get a subscription |
| `PUT` | `/api/subscriptions/{token}` | Update a subscription |
| `POST` | `/api/subscriptions/{token}/pause` | Pause a subscription |
| `POST` | `/api/subscriptions/{token}/resume` | Resume a subscription |
| `POST` | `/api/subscriptions/{token}/cancel` | Cancel a subscription |
| `POST` | `/api/subscriptions/{token}/reactivate` | Reactivate a subscription |

## Subscription Lifecycle

Every subscription moves through a set of states that reflect its current billing and fulfillment status. Understanding these states helps you build accurate UI and automate the right workflows.

| State | Description |
|-------|-------------|
| `active` | The subscription is running normally. Recurring orders are processed on schedule. |
| `paused` | The subscription is temporarily suspended. No orders are processed until it is resumed. |
| `cancelled` | The subscription has been terminated. It cannot receive new orders. Use `reactivate` to restore it. |

Transition between states using the lifecycle action endpoints. For example, send a `POST` to `/api/subscriptions/{token}/pause` to pause an active subscription, and `POST` to `/api/subscriptions/{token}/resume` to make it active again.

## Subscription Plans

Subscriptions are built on subscription plans that define the billing interval, pricing, and included products. Retrieve all available plans for your account with:

```bash
curl https://api.fluid.app/api/subscription_plans \
  -H "Authorization: Bearer <your_token>"
```

Each plan includes an `id`, `name`, `interval` (e.g., `monthly`, `weekly`), `interval_count`, and the associated product or variant configuration. Reference the plan's `id` when creating a new subscription.

## Create a Subscription

Send a `POST` request to `/api/subscriptions` to enroll a customer in a subscription plan. Provide the `customer_id`, the `subscription_plan_id`, and the payment method to charge on each billing cycle.

```bash
curl -X POST https://api.fluid.app/api/subscriptions \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "customer_id": 7821,
    "subscription_plan_id": 12,
    "payment_method_id": 55,
    "next_order_date": "2025-07-01",
    "shipping_address": {
      "first_name": "Jane",
      "last_name": "Smith",
      "line1": "123 Main St",
      "city": "Austin",
      "state": "TX",
      "postal_code": "78701",
      "country": "US"
    }
  }'
```

<ParamField body="customer_id" type="integer" required>
  The ID of the customer to enroll in the subscription.
</ParamField>

<ParamField body="subscription_plan_id" type="integer" required>
  The ID of the subscription plan that defines the billing interval and products.
</ParamField>

<ParamField body="payment_method_id" type="integer" required>
  The ID of the customer's saved payment method to charge on each billing cycle.
</ParamField>

<ParamField body="next_order_date" type="string">
  ISO 8601 date for the first (or next) order. Defaults to today if omitted.
</ParamField>

<ParamField body="shipping_address" type="object" required>
  The shipping address for recurring order deliveries.
</ParamField>

The response includes the subscription `token`, which you use as the path parameter for all subsequent subscription operations.
