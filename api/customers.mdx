The Customers API lets you create and manage customer profiles in your Fluid storefront. Use it to look up customers by ID, update contact information, manage saved payment methods and shipping addresses, view order history, and interact with the loyalty points ledger. All customer endpoints require a valid company or partner token.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/customers` | List customers |
| `POST` | `/api/customers` | Create a customer |
| `GET` | `/api/customers/{id}` | Get a customer |
| `PATCH` | `/api/customers/{id}` | Update a customer |
| `DELETE` | `/api/customers/{id}` | Delete a customer |
| `GET` | `/api/customers/stats` | Customer statistics |
| `GET` | `/api/customers/{id}/payment_methods` | List payment methods |
| `POST` | `/api/customers/{id}/payment_methods` | Add payment method |
| `PATCH` | `/api/customers/{id}/payment_methods/{pm_id}` | Update payment method |
| `DELETE` | `/api/customers/{id}/payment_methods/{pm_id}` | Delete payment method |
| `GET` | `/api/v202506/customers/{id}/orders` | Customer's orders |
| `GET` | `/api/v202506/customers/{id}/points_ledgers` | List points ledger |
| `POST` | `/api/v202506/customers/{id}/points_ledgers` | Add points entry |

## List Customers

Retrieve a paginated list of all customers in your account. Filter by email or name using the `search` parameter.

```bash
curl "https://api.fluid.app/api/customers?page=1&per_page=20&search=jane" \
  -H "Authorization: Bearer <your_token>"
```

<ParamField query="page" type="integer">
  Page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of customers per page. Defaults to `20`.
</ParamField>

<ParamField query="search" type="string">
  Filter by name or email address. Returns partial matches.
</ParamField>

## Customer Fields

<ResponseField name="id" type="integer">
  Unique numeric identifier for the customer.
</ResponseField>

<ResponseField name="email" type="string">
  Customer's email address. Must be unique within your account.
</ResponseField>

<ResponseField name="first_name" type="string">
  Customer's first name.
</ResponseField>

<ResponseField name="last_name" type="string">
  Customer's last name.
</ResponseField>

<ResponseField name="phone" type="string">
  Customer's phone number in E.164 format (e.g., `"+15555550100"`).
</ResponseField>

<ResponseField name="addresses" type="array">
  Array of saved address objects. Each address includes `line1`, `line2`, `city`, `state`, `postal_code`, `country`, and an `is_default` flag.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the customer record was created.
</ResponseField>

## Payment Methods

Retrieve or add saved payment methods for a customer. Payment methods are stored securely and can be referenced when creating orders or subscriptions for that customer.

**List payment methods for a customer:**

```bash
curl https://api.fluid.app/api/customers/7821/payment_methods \
  -H "Authorization: Bearer <your_token>"
```

**Add a payment method:**

```bash
curl -X POST https://api.fluid.app/api/customers/7821/payment_methods \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_token": "tok_visa_4242",
    "set_as_default": true
  }'
```

<ParamField body="payment_token" type="string" required>
  A tokenized payment reference generated by your payment processor. Never pass raw card numbers.
</ParamField>

<ParamField body="set_as_default" type="boolean">
  If `true`, sets this payment method as the customer's default for future transactions. Defaults to `false`.
</ParamField>

## Points Ledger

The points ledger tracks a customer's loyalty points balance. Use the ledger endpoint to view the full transaction history, including points earned from orders and any manual adjustments.

**View a customer's points ledger:**

```bash
curl https://api.fluid.app/api/v202506/customers/7821/points_ledgers \
  -H "Authorization: Bearer <your_token>"
```

Each ledger entry includes a `type` (`credit` or `debit`), a `points` amount, a human-readable `description`, and a `created_at` timestamp. The response also includes the customer's current `balance` in the `meta` object.
