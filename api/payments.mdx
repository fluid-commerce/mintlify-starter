The Payments API lets you tokenize payment methods, process charges through connected payment accounts, retrieve and filter transaction history, and configure intelligent payment routing across multiple processors. Use it to build checkout flows, reconcile transactions, and optimize authorization rates through Fluid's orchestration engine.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v202506/payments/tokenize_card` | Tokenize a card |
| `POST` | `/api/v202506/payments/{payment_account_id}` | Process a payment |
| `GET` | `/api/v202506/transactions` | List transactions |
| `GET` | `/api/v202506/transactions/filter_options` | Transaction filter options |
| `PATCH` | `/api/payments/{payment_uuid}` | Update a payment |
| `GET` | `/api/payments/{payment_uuid}` | Get a payment |
| `POST` | `/api/v202506/payments/settle` | Settle a payment |
| `PATCH` | `/api/v202506/payments/cancel/{order_token}` | Cancel a payment |

## Tokenize a Card

Before charging a card, tokenize it to receive a secure token that represents the card data. Tokenization keeps raw card numbers off your servers and reduces your PCI scope. Send the card details once to receive a `tokenized_card` object you can reuse for processing.

```bash
curl -X POST https://api.fluid.app/api/v202506/payments/tokenize_card \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "card": {
      "number": "4111111111111111",
      "expiration_month": "12",
      "expiration_year": "2027",
      "cvv": "123",
      "cardholder_name": "Jane Smith"
    }
  }'
```

**Response**

```json
{
  "tokenized_card": {
    "token": "tok_01HXYZ1234567890ABCDEFGH",
    "last_four": "1111",
    "brand": "visa",
    "expiration_month": "12",
    "expiration_year": "2027",
    "cardholder_name": "Jane Smith"
  },
  "meta": {
    "request_id": "req_01HXYZ",
    "timestamp": "2025-06-01T12:00:00Z"
  }
}
```

## Process a Payment

Submit a payment using a tokenized card and a target payment account. Specify the `payment_account_id` in the URL path to route the charge to the correct processor. Include the tokenized card data and the amount in the request body.

```bash
curl -X POST https://api.fluid.app/api/v202506/payments/acc_01HXYZ \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment": {
      "tokenized_card": {
        "token": "tok_01HXYZ1234567890ABCDEFGH",
        "last_four": "1111",
        "brand": "visa",
        "expiration_month": "12",
        "expiration_year": "2027",
        "cardholder_name": "Jane Smith"
      },
      "amount": 4999,
      "currency": "USD",
      "order_token": "order_abc123",
      "billing_address": {
        "first_name": "Jane",
        "last_name": "Smith",
        "address1": "123 Main St",
        "city": "San Francisco",
        "state": "CA",
        "zip": "94105",
        "country": "US"
      }
    }
  }'
```

<Note>
  `amount` is expressed in the smallest currency unit (cents for USD). A value of `4999` represents $49.99.
</Note>

**Response**

```json
{
  "payment": {
    "uuid": "pay_01HXYZ",
    "status": "approved",
    "amount": 4999,
    "currency": "USD",
    "order_token": "order_abc123",
    "processor": "stripe",
    "authorization_code": "AUTH123"
  },
  "meta": {
    "request_id": "req_01HABC",
    "timestamp": "2025-06-01T12:00:05Z"
  }
}
```

### Settle a Payment

Capture a previously authorized payment by calling the settle endpoint with the payment UUID.

```bash
curl -X POST https://api.fluid.app/api/v202506/payments/settle \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_uuid": "pay_01HXYZ",
    "amount": 4999
  }'
```

### Cancel a Payment

Cancel a pending or authorized payment using the order token.

```bash
curl -X PATCH https://api.fluid.app/api/v202506/payments/cancel/order_abc123 \
  -H "Authorization: Bearer <token>"
```

## Transactions

Retrieve a paginated list of all transactions for your account. Use query parameters to filter by date range, status, processor, and more.

```bash
curl "https://api.fluid.app/api/v202506/transactions?status=approved&from=2025-05-01&to=2025-05-31&page=1&per_page=25" \
  -H "Authorization: Bearer <token>"
```

**Common query parameters**

<ParamField query="status" type="string">
  Filter by transaction status. Common values: `approved`, `declined`, `refunded`, `pending`.
</ParamField>

<ParamField query="from" type="string">
  Start date for the query range in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="to" type="string">
  End date for the query range in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="payment_account_id" type="string">
  Limit results to a specific payment account (processor).
</ParamField>

<ParamField query="order_token" type="string">
  Retrieve transactions associated with a specific order.
</ParamField>

<ParamField query="page" type="integer">
  Page number for pagination. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of records per page. Defaults to `25`, maximum `100`.
</ParamField>

### Transaction Filter Options

Fetch the available filter values (statuses, processors, currency codes) dynamically to power filter UIs.

```bash
curl https://api.fluid.app/api/v202506/transactions/filter_options \
  -H "Authorization: Bearer <token>"
```

## Payment Routing (Fluid Orchestration)

Fluid's orchestration engine intelligently routes payments across your connected processors. You can split traffic by volume percentage, define rules based on payment method, order value, or geography, and configure fallback processors for failed authorizations. The orchestration layer sits between your checkout and your processors — no code changes needed when you adjust routing logic.

### Orchestration Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/fluid_orchestration/status` | Current routing configuration |
| `POST` | `/api/fluid_orchestration/configurations` | Create a routing configuration |
| `PATCH` | `/api/fluid_orchestration/configurations/{id}` | Update a configuration |
| `DELETE` | `/api/fluid_orchestration/configurations/{id}` | Delete a configuration (must deactivate first) |
| `POST` | `/api/fluid_orchestration/configurations/{id}/activate` | Activate a configuration |
| `POST` | `/api/fluid_orchestration/configurations/{id}/deactivate` | Deactivate a configuration |
| `GET` | `/api/fluid_orchestration/default_fallback` | View fallback processor order |
| `PATCH` | `/api/fluid_orchestration/default_fallback` | Update fallback order |

### View Current Routing Status

```bash
curl https://api.fluid.app/api/fluid_orchestration/status \
  -H "Authorization: Bearer <token>"
```

### Create a Volume-Based Routing Configuration

Route 70% of traffic to one processor and 30% to another. Fluid distributes payments stochastically across the split at runtime.

```bash
curl -X POST https://api.fluid.app/api/fluid_orchestration/configurations \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "configuration": {
      "name": "70/30 Volume Split",
      "configuration_type": "volume_based",
      "distributions": [
        {
          "payment_account_id": "acc_stripe_01",
          "percentage": 70
        },
        {
          "payment_account_id": "acc_braintree_01",
          "percentage": 30
        }
      ]
    }
  }'
```

### Create a Rule-Based Routing Configuration

Route Apple Pay transactions to one processor and standard card payments to another. Rules are evaluated in priority order — the first match wins. Conditions can filter on `payment_source` (`card`, `apple_pay`, `google_pay`), `country_iso` (array of ISO codes), `amount_greater_than`, `amount_less_than`, and `currency_code`.

```bash
curl -X POST https://api.fluid.app/api/fluid_orchestration/configurations \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "configuration": {
      "name": "Method-Based Routing",
      "configuration_type": "rule_based",
      "rules": [
        {
          "priority": 1,
          "payment_account_id": "acc_stripe_01",
          "conditions": {
            "payment_source": "apple_pay"
          }
        },
        {
          "priority": 2,
          "payment_account_id": "acc_braintree_01",
          "conditions": {
            "payment_source": "card"
          }
        }
      ]
    }
  }'
```

### Activate a Configuration

```bash
curl -X POST https://api.fluid.app/api/fluid_orchestration/configurations/cfg_01HXYZ/activate \
  -H "Authorization: Bearer <token>"
```

### Update the Default Fallback Order

When a processor declines or is unavailable, Fluid retries the payment using the fallback order. Update the priority list by providing an ordered array of payment account IDs.

```bash
curl -X PATCH https://api.fluid.app/api/fluid_orchestration/default_fallback \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "fallback_order": ["acc_stripe_01", "acc_braintree_01", "acc_paypal_01"]
  }'
```

## Reports

Pull aggregated payment metrics for dashboards and reconciliation. All report endpoints accept `from` and `to` query parameters in `YYYY-MM-DD` format.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v202506/payments/reports/transaction_volume` | Total transaction volume over a period |
| `GET` | `/api/v202506/payments/reports/transaction_amount` | Total transaction amount over a period |
| `GET` | `/api/v202506/payments/reports/average_latency` | Average payment processing latency |
| `GET` | `/api/v202506/payments/reports/average_latency_by_provider` | Average latency broken down by provider |
| `GET` | `/api/v202506/payments/reports/approval_rate` | Approval rate breakdown by processor |
| `GET` | `/api/v202506/payments/reports/error_rate` | Error rate breakdown by processor |
| `GET` | `/api/v202506/payments/reports/kpis` | Key payment performance indicators |
| `GET` | `/api/v202506/payments/reports/available_providers` | List of providers available for reporting |

```bash
curl "https://api.fluid.app/api/v202506/payments/reports/approval_rate?from=2025-05-01&to=2025-05-31" \
  -H "Authorization: Bearer <token>"
```
