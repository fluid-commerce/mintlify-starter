The Fluid Commerce API is a RESTful API that gives you full programmatic control over your We-Commerce storefront. Use it to manage your product catalog, process orders, handle customer accounts, configure payments, trigger webhooks, and orchestrate integrations — all from your own systems or custom-built applications.

## Base URLs

All API requests are made over HTTPS. Use the production URL for live traffic, or the mock server to explore and test endpoints without affecting real data.

| Environment | Base URL |
|-------------|----------|
| Production | `https://api.fluid.app/` |
| Mock server | `https://docs.fluid.app/_mock/docs/apis/swagger/` |

## Authentication

Every request to the Fluid API must include a valid bearer token in the `Authorization` header.

```bash
Authorization: Bearer <your_token>
```

Fluid supports multiple token types — company tokens, partner tokens, and public tokens — each with different scopes and intended use cases. See the [Authentication guide](/api/authentication) for full details on generating and managing tokens.

## Request Format

The API accepts JSON-encoded request bodies. Always set the `Content-Type` header when sending a request body.

```bash
Content-Type: application/json
```

Send all request bodies as valid JSON. String values must be quoted, boolean values must be `true` or `false`, and numeric values must not be quoted.

## Response Format

All API responses are JSON-encoded and include a `meta` object at the top level. The `meta` object contains a unique `request_id` for support and tracing, a `timestamp` for the response time, and — on list endpoints — a `pagination` object.

```json
{
  "meta": {
    "request_id": "123e4567-e89b-12d3-a456-426614174000",
    "timestamp": "2021-01-01T00:00:00Z",
    "pagination": {
      "current_page": 1,
      "total_pages": 5,
      "per_page": 20,
      "total_count": 100
    }
  }
}
```

## Pagination

Most list endpoints support pagination via query parameters. Use `page` to specify which page to retrieve and `per_page` to control how many records are returned per request.

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | integer | Page number to retrieve (default: `1`) |
| `per_page` | integer | Number of records per page (default: `20`) |

The `meta.pagination` object in each list response tells you the current page, total pages, and total record count so you can iterate through results programmatically.

## API Groups

The Fluid API is organized into functional groups. Each group covers a distinct area of your commerce operations.

| Group | Resources |
|-------|-----------|
| **Commerce** | Products, Orders, Carts, Customers, Discounts, Subscriptions |
| **Payments** | Payment accounts, transactions, routing |
| **Integrations** | Webhooks, Drop Zones, Droplets |
| **Users** | Reps, Admins, Users |
| **Content** | Media, Pages, Posts, Libraries |
| **Settings** | Brand guidelines, display settings, languages, warehouses |
