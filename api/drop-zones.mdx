Drop Zones let you inject custom iframe content at defined positions within the Fluid checkout experience. You configure a Drop Zone with an embed URL and a target location, and Fluid renders it as an iframe at that position for every shopper who passes through. Supported locations include the checkout page, order confirmation screen, subscription flows, and product pages. Drop Zones are the primary mechanism [Droplets](/concepts/droplets) use to embed custom UI directly inside the checkout flow.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/drop_zones` | List drop zones |
| `POST` | `/api/drop_zones` | Create a drop zone |
| `GET` | `/api/drop_zones/{uuid}` | Get a drop zone |
| `PUT` | `/api/drop_zones/{uuid}` | Update a drop zone |
| `DELETE` | `/api/drop_zones/{uuid}` | Delete a drop zone |
| `GET` | `/api/drop_zones/constants` | Available drop zone locations |

## Create a Drop Zone

Register a new Drop Zone by providing a name, an embed URL, and optionally the target location and additional settings. Your embed URL must use `https://`.

```bash
curl -X POST https://api.fluid.app/api/drop_zones \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "drop_zone": {
      "name": "My Custom Widget",
      "embed_url": "https://my-app.com/widget",
      "active": true
    }
  }'
```

**Response**

```json
{
  "drop_zone": {
    "uuid": "dz_01HXYZ1234567890",
    "name": "My Custom Widget",
    "embed_url": "https://my-app.com/widget",
    "active": true,
    "settings": {}
  },
  "meta": {
    "request_id": "req_01HABC",
    "timestamp": "2025-06-01T12:00:05Z"
  }
}
```

<ParamField body="name" type="string" required>
  A descriptive name for the Drop Zone, used to identify it in the Fluid dashboard.
</ParamField>

<ParamField body="embed_url" type="string" required>
  The HTTPS URL Fluid loads inside the iframe. Must be publicly accessible and served over `https://`.
</ParamField>

<ParamField body="active" type="boolean">
  Whether the Drop Zone is rendered to shoppers immediately. Defaults to `false`.
</ParamField>

<ParamField body="settings" type="object">
  Optional additional configuration, such as target zone position or iframe dimensions.
</ParamField>

## List Drop Zones

Retrieve all Drop Zones configured for your company. Use query parameters to filter by page type, active status, or visibility.

```bash
curl "https://api.fluid.app/api/drop_zones?active=true&page_type=checkout&page=1&per_page=25" \
  -H "Authorization: Bearer <token>"
```

<ParamField query="active" type="boolean">
  Filter by active status. Pass `true` to return only active Drop Zones.
</ParamField>

<ParamField query="page_type" type="string">
  Filter by page type (e.g. `checkout`, `order_confirmation`, `product`). See `GET /api/drop_zones/constants` for valid values.
</ParamField>

<ParamField query="public" type="boolean">
  Filter by public visibility.
</ParamField>

<ParamField query="page" type="integer">
  Page number for pagination. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of records per page.
</ParamField>

## Get a Drop Zone

Fetch a single Drop Zone by its UUID.

```bash
curl https://api.fluid.app/api/drop_zones/dz_01HXYZ1234567890 \
  -H "Authorization: Bearer <token>"
```

---

## Update a Drop Zone

Update any fields on an existing Drop Zone using `PUT`. Include the full resource body.

```bash
curl -X PUT https://api.fluid.app/api/drop_zones/dz_01HXYZ1234567890 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "drop_zone": {
      "name": "My Custom Widget (v2)",
      "embed_url": "https://my-app.com/widget-v2",
      "active": true
    }
  }'
```

---

## Delete a Drop Zone

Remove a Drop Zone by UUID. Deleted Drop Zones are immediately removed from the checkout flow for all shoppers.

```bash
curl -X DELETE https://api.fluid.app/api/drop_zones/dz_01HXYZ1234567890 \
  -H "Authorization: Bearer <token>"
```

## Drop Zone Object

<ResponseField name="uuid" type="string">
  Unique identifier for the Drop Zone, used in all subsequent requests.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the Drop Zone as configured by you.
</ResponseField>

<ResponseField name="active" type="boolean">
  Whether the Drop Zone is currently active and visible to shoppers in the checkout flow.
</ResponseField>

<ResponseField name="embed_url" type="string">
  The `https://` URL loaded inside the iframe at the designated checkout position.
</ResponseField>

<ResponseField name="settings" type="object">
  Additional configuration for the Drop Zone, such as target zone position, iframe height, and scroll behavior.
</ResponseField>

---

<Note>
  Drop Zones are the primary way [Droplets](/concepts/droplets) embed custom UI inside the Fluid checkout. If you are building a Droplet, create a Drop Zone and reference it in your Droplet's configuration so it is automatically injected when your Droplet is installed.
</Note>
