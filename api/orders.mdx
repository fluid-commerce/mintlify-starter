The Orders API lets you retrieve, update, and act on orders placed through your Fluid storefront. Use it to build order management dashboards, automate fulfillment workflows, sync orders to third-party logistics systems, and issue refunds. All order endpoints require a valid company or partner token.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v202506/orders` | List orders |
| `GET` | `/api/v202506/orders/stats` | Get order statistics |
| `GET` | `/api/v202506/orders/{id}` | Get an order |
| `PATCH` | `/api/orders/{id}` | Update an order |
| `GET` | `/api/order_fulfillments` | List fulfillments |
| `POST` | `/api/order_fulfillments` | Create a fulfillment |
| `PUT` | `/api/order_fulfillments/{id}` | Update a fulfillment |
| `POST` | `/api/refunds` | Issue a refund |
| `POST` | `/api/refunds/items` | Refund specific items |

## List Orders

Retrieve a paginated list of orders. Use the `page` and `per_page` parameters to iterate through large result sets.

```bash
curl "https://api.fluid.app/api/v202506/orders?page=1&per_page=20" \
  -H "Authorization: Bearer <your_token>"
```

<ParamField query="page" type="integer">
  Page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of orders per page. Defaults to `20`.
</ParamField>

## Order Response Fields

<ResponseField name="id" type="integer">
  Unique numeric identifier for the order.
</ResponseField>

<ResponseField name="order_number" type="string">
  Human-readable order reference shown to customers (e.g., `"ORD-10042"`).
</ResponseField>

<ResponseField name="status" type="string">
  Current order status. Common values: `pending`, `processing`, `fulfilled`, `cancelled`, `refunded`.
</ResponseField>

<ResponseField name="customer" type="object">
  Customer information associated with the order, including `id`, `email`, `first_name`, and `last_name`.
</ResponseField>

<ResponseField name="line_items" type="array">
  Array of ordered items. Each item includes `product_id`, `variant_id`, `quantity`, `price`, and `sku`.
</ResponseField>

<ResponseField name="total_price" type="string">
  Formatted total order amount including taxes and shipping (e.g., `"89.97"`).
</ResponseField>

<ResponseField name="currency" type="string">
  ISO 4217 currency code for the order (e.g., `"USD"`).
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the order was placed.
</ResponseField>

## Fulfillments

Create a fulfillment to mark one or more line items in an order as shipped. Include a `tracking_number` and `tracking_company` so customers can track their shipment.

**Create a fulfillment:**

```bash
curl -X POST https://api.fluid.app/api/order_fulfillments \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": 10042,
    "tracking_number": "1Z999AA10123456784",
    "tracking_company": "UPS",
    "line_items": [
      { "id": 5001, "quantity": 1 },
      { "id": 5002, "quantity": 2 }
    ],
    "notify_customer": true
  }'
```

<ParamField body="order_id" type="integer" required>
  The ID of the order being fulfilled.
</ParamField>

<ParamField body="tracking_number" type="string">
  Carrier tracking number for this shipment.
</ParamField>

<ParamField body="tracking_company" type="string">
  Name of the shipping carrier (e.g., `"UPS"`, `"FedEx"`, `"USPS"`).
</ParamField>

<ParamField body="line_items" type="array" required>
  Array of `{ id, quantity }` objects specifying which line items — and how many units — to include in this fulfillment.
</ParamField>

<ParamField body="notify_customer" type="boolean">
  If `true`, Fluid sends a shipment notification email to the customer. Defaults to `false`.
</ParamField>

Use `PUT /api/order_fulfillments/{id}` to update an existing fulfillment — for example, to correct a tracking number after the fulfillment was created.

## Refunds

Issue a full or partial refund for an order using the refund endpoints. To refund the entire order, send a `POST` to `/api/refunds` with the `order_id`. To refund specific line items and quantities, use `/api/refunds/items`.

**Issue a full refund:**

```bash
curl -X POST https://api.fluid.app/api/refunds \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": 10042,
    "reason": "Customer request"
  }'
```

**Refund specific items:**

```bash
curl -X POST https://api.fluid.app/api/refunds/items \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": 10042,
    "line_items": [
      { "id": 5001, "quantity": 1, "amount": "29.99" }
    ],
    "reason": "Item arrived damaged"
  }'
```

<Warning>
  Refunds are irreversible once processed. Always confirm the order ID and line item details before submitting a refund request.
</Warning>
