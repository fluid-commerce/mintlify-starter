The Droplets API enables third-party developers to create, configure, and manage Droplets — packaged integrations that merchants install from the Fluid Marketplace. Use this API to register your Droplet, provide embed URLs that appear in the Fluid admin, configure lifecycle webhooks that fire when a company installs or uninstalls your Droplet, and manage individual installations. You can also list every company currently using your Droplet to support provisioning, billing, and support workflows.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/droplets` | List droplets |
| `POST` | `/api/droplets` | Create a droplet |
| `GET` | `/api/droplets/{uuid}` | Get a droplet |
| `PUT` | `/api/droplets/{uuid}` | Update a droplet |
| `DELETE` | `/api/droplets/{uuid}` | Delete a droplet |
| `GET` | `/api/droplets/{uuid}/companies` | List companies using droplet |
| `GET` | `/api/droplet_installations` | List installations |
| `POST` | `/api/droplet_installations` | Create an installation (set scopes) |
| `GET` | `/api/droplet_installations/{uuid}` | Get an installation |
| `PUT` | `/api/droplet_installations/{uuid}` | Update an installation (change scopes) |
| `DELETE` | `/api/droplet_installations/{uuid}` | Delete an installation |
| `POST` | `/api/droplet_installations/exchange` | Exchange install token for auth credentials |

## Create a Droplet

Register a new Droplet by providing your embed URL, lifecycle webhook URLs, and marketplace metadata. Fluid calls your `install_webhook_url` whenever a company installs your Droplet and your `uninstall_webhook_url` when they remove it — use these to provision or deprovision the company in your own system.

```bash
curl -X POST https://api.fluid.app/api/droplets \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "droplet": {
      "name": "My Droplet",
      "embed_url": "https://my-company.com/embed",
      "install_webhook_url": "https://my-company.com/webhooks/install",
      "uninstall_webhook_url": "https://my-company.com/webhooks/uninstall",
      "active": true,
      "settings": {
        "marketplace_page": {
          "title": "My Droplet",
          "summary": "A short description",
          "logo_url": "https://my-company.com/logo.svg"
        }
      }
    }
  }'
```

<ParamField body="name" type="string" required>
  The internal name of your Droplet. Also displayed to merchants unless overridden in `settings.marketplace_page.title`.
</ParamField>

<ParamField body="embed_url" type="string" required>
  The base URL Fluid loads inside the Fluid admin when a company has your Droplet installed. Fluid appends the installing company's ID as a query parameter (e.g. `?company_id=123`) so your embed can scope its UI accordingly.
</ParamField>

<ParamField body="install_webhook_url" type="string">
  Fluid sends a `POST` to this URL when a company installs your Droplet. Use it to create accounts, configure resources, or trigger onboarding flows in your system.
</ParamField>

<ParamField body="uninstall_webhook_url" type="string">
  Fluid sends a `POST` to this URL when a company uninstalls your Droplet. Use it to clean up resources and update billing.
</ParamField>

<ParamField body="active" type="boolean">
  Whether your Droplet is discoverable. Set to `true` once you are ready for merchants to find and install it. Pending Marketplace approval is also required — see the note below.
</ParamField>

<ParamField body="settings" type="object">
  Additional Droplet configuration. The `marketplace_page` key controls how your Droplet appears in the Fluid Marketplace listing, including title, summary, and logo URL.
</ParamField>

**Response**

```json
{
  "droplet": {
    "uuid": "drp_01HXYZ1234567890",
    "name": "My Droplet",
    "embed_url": "https://my-company.com/embed",
    "install_webhook_url": "https://my-company.com/webhooks/install",
    "uninstall_webhook_url": "https://my-company.com/webhooks/uninstall",
    "active": true,
    "settings": {
      "marketplace_page": {
        "title": "My Droplet",
        "summary": "A short description",
        "logo_url": "https://my-company.com/logo.svg"
      }
    }
  },
  "meta": {
    "request_id": "req_01HABC",
    "timestamp": "2025-06-01T12:00:00Z"
  }
}
```

## Droplet Object

<ResponseField name="uuid" type="string">
  Unique identifier for the Droplet. Use this in all subsequent requests and when referencing the Droplet in Drop Zone configurations.
</ResponseField>

<ResponseField name="name" type="string">
  The name of the Droplet, shown in the Fluid admin and Marketplace.
</ResponseField>

<ResponseField name="embed_url" type="string">
  The base URL Fluid loads in the admin for companies that have installed your Droplet. Fluid appends the company ID as a query parameter automatically.
</ResponseField>

<ResponseField name="install_webhook_url" type="string">
  The URL Fluid calls when a company installs your Droplet.
</ResponseField>

<ResponseField name="uninstall_webhook_url" type="string">
  The URL Fluid calls when a company uninstalls your Droplet.
</ResponseField>

<ResponseField name="active" type="boolean">
  Whether the Droplet is active and discoverable in the Marketplace (subject to Fluid approval).
</ResponseField>

## List Companies Using Your Droplet

Retrieve a paginated list of all companies that currently have your Droplet installed. This is useful for support, billing reconciliation, and scoping feature rollouts.

```bash
curl https://api.fluid.app/api/droplets/drp_01HXYZ1234567890/companies \
  -H "Authorization: Bearer <token>"
```

## Manage Installations

An installation record is created each time a company installs your Droplet. You can list all installations, retrieve a specific one, update installation-level settings, or delete an installation to forcibly remove a company's access.

```bash
# List all installations
curl https://api.fluid.app/api/droplet_installations \
  -H "Authorization: Bearer <token>"

# Get a specific installation
curl https://api.fluid.app/api/droplet_installations/ins_01HXYZ \
  -H "Authorization: Bearer <token>"

# Update an installation
curl -X PUT https://api.fluid.app/api/droplet_installations/ins_01HXYZ \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "droplet_installation": {
      "active": false
    }
  }'

# Delete an installation
curl -X DELETE https://api.fluid.app/api/droplet_installations/ins_01HXYZ \
  -H "Authorization: Bearer <token>"
```

## The Installation Webhook Flow

When a merchant installs your Droplet, Fluid immediately sends a `POST` request to your `install_webhook_url` containing the company's details and the installation UUID. Your endpoint should respond with `200 OK` within 10 seconds to confirm receipt. If the response times out or returns an error, Fluid will retry delivery with exponential backoff.

The uninstall webhook follows the same pattern — Fluid fires your `uninstall_webhook_url` as soon as the merchant removes the Droplet.

### Exchanging the Install Token for Credentials

After receiving the install webhook, exchange the `exchange_token` (prefixed `dex_`) it contains for your long-lived API credentials by calling the exchange endpoint. Fluid returns two tokens you must store securely per installation.

```bash
curl -X POST https://api.fluid.app/api/droplet_installations/exchange \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "exchange_token": "dex_<one_time_token_from_webhook>"
  }'
```

**Response**

```json
{
  "authentication_token": "dit_01HXYZ1234567890ABCDEFGH",
  "webhook_verification_token": "wvt_01HXYZ1234567890ABCDEFGH"
}
```

<ResponseField name="authentication_token" type="string">
  A long-lived API token (prefixed `dit_`) used to authenticate requests your Droplet makes to the Fluid API on behalf of the installing company. Store this securely — it will not be returned again.
</ResponseField>

<ResponseField name="webhook_verification_token" type="string">
  A token (prefixed `wvt_`) used to verify that incoming webhook payloads originated from Fluid. Include it in HMAC signature verification for all webhooks you receive from Fluid for this installation.
</ResponseField>

For a complete walkthrough of handling these webhooks, verifying the payloads, and building the full install/uninstall flow, see the [Creating Droplets guide](/guides/creating-droplets).
