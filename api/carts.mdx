The Carts API lets you create and manage shopping carts programmatically. Use it to build server-side cart workflows, apply discount codes, set shipping addresses, and manage cart metadata. Each cart is identified by a unique `cart_token` that persists across requests and can be passed to the FairShare SDK to drive client-side checkout flows.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/carts` | Create a cart |
| `GET` | `/api/carts/{cart_token}` | Get a cart |
| `PATCH` | `/api/carts/{cart_token}` | Update a cart |
| `POST` | `/api/carts/{cart_token}/address` | Set cart address |
| `POST` | `/api/carts/{cart_token}/discount` | Apply discount |
| `DELETE` | `/api/carts/{cart_token}/discount` | Remove discount |
| `PATCH` | `/api/carts/{cart_token}/append_metadata` | Add metadata |
| `PATCH` | `/api/carts/{cart_token}/remove_metadata` | Remove metadata |

## Create a Cart

Send a `POST` request to `/api/carts` to create a new cart. You can optionally include initial line items, a customer reference, or rep attribution data at creation time.

```bash
curl -X POST https://api.fluid.app/api/carts \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "customer_id": 7821,
    "attribution": {
      "rep_id": 334
    }
  }'
```

The response includes the `cart_token` you will use in all subsequent requests for this cart.

## Cart Response Fields

<ResponseField name="cart_token" type="string">
  Unique string identifier for the cart. Pass this token in the URL path for all cart operations and to the FairShare SDK for client-side checkout.
</ResponseField>

<ResponseField name="items" type="array">
  Array of line item objects in the cart. Each item includes `variant_id`, `quantity`, `price`, and `product_title`.
</ResponseField>

<ResponseField name="total_price" type="string">
  Formatted cart total including any applied discounts (e.g., `"59.97"`).
</ResponseField>

<ResponseField name="discount_code" type="string or null">
  The discount code currently applied to the cart, or `null` if no discount is active.
</ResponseField>

<ResponseField name="attribution" type="object">
  Rep attribution data associated with this cart. Includes `rep_id` and any additional attribution metadata used for commission tracking.
</ResponseField>

## Adding Items

For client-side cart interactions — such as letting a shopper add items from a product page — use the FairShare SDK, which manages the cart session and communicates with the Carts API on your behalf. See the [Cart API SDK reference](/sdk/cart-api) for setup instructions.

For server-side cart management, update the cart's line items by sending a `PATCH` request with the updated `items` array:

```bash
curl -X PATCH https://api.fluid.app/api/carts/cart_abc123xyz \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      { "variant_id": 9901, "quantity": 2 },
      { "variant_id": 9905, "quantity": 1 }
    ]
  }'
```

## Applying Discounts

Apply a discount code to a cart by sending a `POST` request to the cart's `/discount` sub-resource. If the code is valid, the cart total updates immediately.

```bash
curl -X POST https://api.fluid.app/api/carts/cart_abc123xyz/discount \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "discount_code": "SUMMER20"
  }'
```

To remove the applied discount code, send a `DELETE` request to the same endpoint:

```bash
curl -X DELETE https://api.fluid.app/api/carts/cart_abc123xyz/discount \
  -H "Authorization: Bearer <your_token>"
```

<Note>
  Cart tokens are the primary interface between the server-side Carts API and the client-side FairShare SDK. Once you create a cart server-side, pass its `cart_token` to the SDK to enable seamless client-side item management, address entry, and checkout. See the [Cart API SDK reference](/sdk/cart-api) for integration details.
</Note>
