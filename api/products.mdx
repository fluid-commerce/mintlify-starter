The Products API gives you full control over your Fluid product catalog. Use it to create new products, update pricing and descriptions, manage variants and options, attach images, duplicate listings, and remove products that are no longer available. All product operations require a valid company or partner token.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/company/v1/products` | List products |
| `POST` | `/api/company/v1/products` | Create a product |
| `GET` | `/api/company/v1/products/{id}` | Get a product |
| `PATCH` | `/api/company/v1/products/{id}` | Update a product |
| `DELETE` | `/api/company/v1/products/{id}` | Delete a product |
| `POST` | `/api/company/v1/products/{id}/duplicate` | Duplicate a product |
| `GET` | `/api/company/v1/variants/{id}` | Get a variant |
| `PATCH` | `/api/company/v1/variants/{id}` | Update a variant |
| `POST` | `/api/company/v1/variants` | Create a variant |

## List Products

Retrieve a paginated list of all products in your catalog. Use the `search` parameter to filter results by name or SKU, and `page`/`per_page` to control pagination.

```bash
curl https://api.fluid.app/api/company/v1/products?page=1&per_page=20&search=shirt \
  -H "Authorization: Bearer <your_token>"
```

<ParamField query="page" type="integer">
  Page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of products per page. Defaults to `20`.
</ParamField>

<ParamField query="search" type="string">
  Filter products by name or SKU. Returns partial matches.
</ParamField>

## Create a Product

Send a `POST` request with a JSON body to create a new product. At minimum, provide a `name`. Include `variants` to set up SKUs, pricing, and inventory at creation time.

```bash
curl -X POST https://api.fluid.app/api/company/v1/products \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Classic Crew Tee",
    "description": "A comfortable everyday crew-neck t-shirt.",
    "variants": [
      {
        "sku": "CCT-SM-WHT",
        "price": "29.99",
        "inventory_quantity": 150,
        "option1": "Small",
        "option2": "White"
      },
      {
        "sku": "CCT-MD-WHT",
        "price": "29.99",
        "inventory_quantity": 200,
        "option1": "Medium",
        "option2": "White"
      }
    ],
    "tags": ["apparel", "basics"]
  }'
```

## Product Response Fields

<ResponseField name="id" type="integer">
  Unique numeric identifier for the product.
</ResponseField>

<ResponseField name="name" type="string">
  The product's display name shown to customers.
</ResponseField>

<ResponseField name="description" type="string">
  Full product description. Supports plain text or HTML.
</ResponseField>

<ResponseField name="price" type="string">
  Formatted price string for the base variant (e.g., `"29.99"`).
</ResponseField>

<ResponseField name="variants" type="array">
  Array of variant objects. Each variant represents a unique SKU with its own pricing, inventory, and option values.
</ResponseField>

<ResponseField name="images" type="array">
  Array of image objects attached to the product. Each image includes a `url`, `position`, and optional `alt` text.
</ResponseField>

<ResponseField name="available" type="boolean">
  `true` if at least one variant is in stock and available for purchase.
</ResponseField>

<ResponseField name="tags" type="array">
  Array of string tags used for filtering and organization.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the product was created.
</ResponseField>

## Variants

Variants represent the individual purchasable versions of a product — for example, different sizes or colors. Every product must have at least one variant. You can create variants at the same time as the product or add them afterward using the standalone variant endpoints.

**Create a variant for an existing product:**

```bash
curl -X POST https://api.fluid.app/api/company/v1/variants \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": 42,
    "sku": "CCT-LG-BLK",
    "price": "29.99",
    "inventory_quantity": 75,
    "option1": "Large",
    "option2": "Black"
  }'
```

**Update an existing variant:**

```bash
curl -X PATCH https://api.fluid.app/api/company/v1/variants/198 \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "price": "24.99",
    "inventory_quantity": 50
  }'
```

Use `PATCH /api/company/v1/products/{id}` to update top-level product fields like `name`, `description`, and `tags`. Use the variant-specific endpoints to update SKU-level data such as pricing, inventory, and option values.
